# Decisions

重要决定按时间追加，不覆盖历史。状态使用 `Accepted`、`Superseded` 或 `Pending`。

## D-001 项目根目录

- 日期：2026-06-20
- 状态：Accepted
- 决定：当前唯一项目路径为 `E:\jielong`。
- 原因：旧移交文档中的 `E:\软件开发\jielong` 已失效，英文路径可减少工具兼容问题。

## D-002 当前只使用模拟外部服务

- 日期：2026-06-20
- 状态：Accepted
- 决定：广告、短信、微信登录和提现均使用模拟实现并保留适配接口。
- 原因：尚未与平台签约，没有正式 SDK 参数和服务端凭证。

## D-003 服务端账本权威

- 日期：2026-06-20
- 状态：Accepted
- 决定：金币到账必须以后端事务成功为准，客户端只展示后端结果。
- 原因：金币可映射提现价值，客户端本地奖励容易造成重复、不同步和伪造。

## D-004 协作模式

- 日期：2026-06-20
- 状态：Accepted
- 决定：Codex 负责全局架构、任务拆解、审查和验收；Claude Code 作为实现副手，只接受小范围任务；必要时使用子 agent 做隔离调查。
- 原因：减少上下文漂移、并行冲突和大范围不可审查改动。

## D-005 完成定义

- 日期：2026-06-20
- 状态：Accepted
- 决定：任务完成必须包含实现、测试或复现证据、Git diff 审查和项目大脑更新。
- 原因：口头说明和页面表象不足以证明系统正确。

## D-006 动态委派策略

- 日期：2026-06-20
- 状态：Accepted
- 决定：不要求固定比例使用 Claude Code。小型精确修复、关键故障和需要连续推理的工作由 Codex 直接处理；跨文件实现、批量代码、重复测试和可并行模块按收益委派给 Claude 或子 agent。
- 原因：Claude 经 Kimi 接口执行小任务通常需要 40 至 120 秒，且可能在测试或回报阶段超时；按总交付时间动态选择更高效。

## D-007 API 地址由构建环境注入

- 日期：2026-06-20
- 状态：Accepted
- 决定：Flutter 使用 `--dart-define=API_BASE_URL=...` 注入后端地址；本机 Web 默认使用 localhost，Android 模拟器 Debug 构建使用 `10.0.2.2`，生产环境必须传入 HTTPS 地址。
- 原因：Android 中 localhost 指向设备自身，硬编码地址无法同时适配 Web、模拟器、真机和生产环境。

## D-008 游戏结算以服务端状态为准

- 日期：2026-06-20
- 状态：Accepted
- 决定：`/game/end` 忽略客户端上传的轮数和成语链，只使用服务端 active game 状态；客户端只能展示服务端返回的金币与纪录。
- 原因：客户端字段可伪造，不能参与具有提现价值的奖励计算。

## D-009 可提现资产禁止客户端直接修改

- 日期：2026-06-20
- 状态：Accepted
- 决定：客户端删除直接增减金币和提示次数的公开方法。游戏、广告、签到、转盘、提示和提现全部等待后端事务成功，再刷新用户资料。尚无后端接口的任务奖励暂不发放。
- 原因：任何本地余额修改都会造成前后端分叉，并可被篡改或重复触发。

## D-010 提现账本语义

- 日期：2026-06-20
- 状态：Accepted
- 决定：提现申请时原子扣减可用金币；拒绝时事务退回；审核支付成功时才增加 `total_withdrawn`。游客不能提现，重复审核必须失败。
- 原因：`total_withdrawn` 表示实际已支付金额，不应包含待审核或已拒绝申请。

## D-011 每日任务由服务端计算和发奖

- 日期：2026-06-20
- 状态：Accepted
- 决定：任务进度从当日游戏、广告和纪录数据实时计算；客户端不提交进度或奖励值。领取记录以 `(user_id, task_key, claim_date)` 唯一键保证每日幂等。
- 原因：客户端任务进度和领取状态可伪造，且并发领取必须由数据库唯一约束兜底。

## D-012 成语词库以共享 JSON 为源并同步数据库

- 日期：2026-06-20
- 状态：Accepted
- 决定：`shared/data/idioms.json` 是词库源数据；游戏服务启动时加载它构建内存索引，管理后台通过 MySQL 管理。每次源数据变更后运行 `npm run import:idioms` 幂等同步数据库。
- 原因：此前游戏拥有 10,010 条本地词库而管理数据库为空，导致后台显示与实际玩法分叉；可重复同步脚本能让新环境恢复一致状态。

## D-013 旧资金接口先以 HTTP 410 下线

- 日期：2026-06-20
- 状态：Accepted
- 决定：旧 `/gold/game-reward` 与 `/gold/use-hint` 保留一版无副作用的 HTTP 410 兼容门，移动端删除调用封装，正式流程只使用 `/game/end` 与 `/game/hint`。
- 原因：直接删除会让旧客户端只得到模糊 404；410 能明确提示迁移，同时立即封死伪造轮数和绕过局内状态的资产风险。

## D-014 提示必须绑定服务端当前链尾并原子扣减

- 日期：2026-06-21
- 状态：Accepted
- 决定：`/game/hint` 仅接受服务端 active game 的当前链尾成语；提示次数使用 `UPDATE ... WHERE hints > 0` 条件更新。
- 原因：只校验游戏归属会允许客户端提交任意成语；先查后扣在并发请求下可能出现负数。
