# Project Brief

## 项目目标

开发一款以成语接龙为核心玩法的休闲 App。用户可通过游戏和模拟激励广告获得金币，达到门槛后提交模拟提现申请；运营方通过管理后台查看用户、金币流水、广告记录和提现申请。

当前阶段只建设可测试、可演示、可继续接入真实平台的产品原型，不接入真实广告、短信、微信登录或自动打款。

## 当前版本边界

- 客户端：Flutter，当前优先验证 Android。
- 后端：Node.js、Express、MySQL。
- 管理后台：React、TypeScript、Ant Design。
- 广告：仅 `MockAdProvider` 语义，模拟完整观看结果。
- 提现：仅创建和审核测试记录，不发生真实资金转移。
- 短信：固定测试验证码，仅限开发环境。
- 外部平台：保留适配接口，签约并取得正式参数后再实现。

## 核心原则

1. 金币余额和流水以后端数据库为唯一事实来源。
2. 客户端不能自行决定可提现金币是否到账。
3. 模拟能力必须有醒目标识，测试数据不得混入未来生产环境。
4. 每次金币变化必须有唯一业务标识、原因和流水记录。
5. 每个任务必须范围小、可测试、可回滚、可通过 Git diff 审查。

## 暂不实施

- 真实广告 SDK 和服务端回调验签。
- 微信、支付宝真实付款或自动提现。
- 真实短信、微信 OAuth。
- 绕过广告平台规则、制造虚假流量或自动刷广告。
- 未经验证的多人对战、邀请裂变和复杂活动系统。

## 成功标准

- 本地可稳定启动客户端、后端和管理后台。
- 模拟广告完成后，由后端原子发放奖励并返回最新余额。
- 金币流水、用户余额、提现冻结或扣减状态始终一致。
- 主要用户路径有自动化测试或可重复执行的验收脚本。
- 新会话只阅读项目大脑文件即可恢复当前工作状态。
