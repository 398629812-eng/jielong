# Roadmap

## 当前阶段：V0.1 稳定测试版

目标：确认现有代码真实可运行，修复前后端不同步，并建立可靠的模拟广告、金币和模拟提现闭环。

### Phase 0：建立基线

- [x] 建立项目大脑文档。
- [x] 审查 Claude Code 历史改动。
- [x] 清理 Git 跟踪边界，确保密钥和依赖目录不入库。
- [x] 记录三端可复现的启动命令和环境版本。

### Phase 1：系统体检

- [x] 后端启动、数据库连接和核心 API 冒烟测试。
- [x] 管理后台生产构建及核心管理员 API 检查。
- [x] Flutter analyze、test 和 Debug APK 构建。
- [ ] Flutter 关键页面视觉与交互巡检。
- [ ] 对照 `docs/API.md` 检查客户端请求与后端响应。

### Phase 2：模拟商业闭环

- [x] 广告完成后由后端确认并发奖。
- [x] 建立广告交易号唯一索引，重复提交不重复发奖。
- [x] 游戏开局、接龙验证和结算接入服务端权威状态。
- [x] 统一任务进度和任务奖励的服务端账本规则。
- [x] 统一签到、转盘、提示和游戏结算的服务端账本规则。
- [x] 模拟提现申请、审核、拒绝和余额处理全部可追溯。
- [ ] 管理后台展示测试数据和测试环境标识。

### Phase 3：体验与质量

- [ ] 修复成语接龙提示和续命边界问题。
- [x] 修复 AppBar 小尺寸倒计时布局溢出。
- [x] 完成金币明细、提现记录、排行榜真实对接。
- [x] 增加关键流程测试、错误状态和空状态。
- [x] 完成 Android 测试 APK。

### Phase 4：真实平台接入准备

- [ ] 定义广告、短信、登录、提现 Provider 接口。
- [ ] 编写接入清单、配置模板和生产安全检查表。
- [ ] 广告平台签约后再确定具体 SDK 和服务端验证方案。

## 发布门槛

任何阶段不得仅凭页面可点击判定完成。必须同时具备：代码审查、测试结果、关键路径演示和文档更新。
