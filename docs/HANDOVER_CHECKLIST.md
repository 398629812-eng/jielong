# 项目交付清单

本清单用于交接给运营公司或服务器负责人。交付时建议双方逐项确认。

## 1. 源码和版本

- Git 仓库地址。
- 当前交付 commit hash。
- `mobile` 客户端源码。
- `backend` 后端源码。
- `admin` 管理后台源码。
- `shared` 成语数据与脚本。
- `docs` 文档目录。

交付时不应包含：

- `.env` 真实环境文件。
- SSH 私钥。
- 云服务 AccessKey。
- 广告平台密钥。
- 短信服务密钥。
- 支付或提现商户密钥。
- 个人手机号、邮箱、身份证、银行卡等信息。

## 2. 服务器资料

由运营公司提供并负责保管：

- 腾讯云账号。
- 服务器公网 IP。
- 服务器登录账号。
- 域名和 DNS 管理权限。
- ICP 备案信息。
- APP 备案信息。

开发者只需要部署所需的临时访问权限。交付完成后，运营方应修改服务器登录密码和密钥。

## 3. 数据库资料

应交付：

- `backend/scripts/init_db.sql`
- 所有 `backend/scripts/migrate_*.sql`
- 数据库初始化说明。
- 数据库备份说明。

不应交付到仓库：

- 生产数据库密码。
- 生产数据库备份文件。
- 用户真实数据导出文件。

## 4. 后端配置

运营方需要在服务器 `.env` 中填写：

- `NODE_ENV`
- `PORT`
- `HOST`
- `TRUST_PROXY`
- `CORS_ORIGINS`
- `DB_HOST`
- `DB_PORT`
- `DB_USER`
- `DB_PASSWORD`
- `DB_NAME`
- `JWT_SECRET`
- `JWT_EXPIRES_IN`
- `ADMIN_JWT_SECRET`
- `ADMIN_USERNAME`

生产环境必须使用强随机密钥。用户 JWT 密钥和管理员 JWT 密钥必须不同。

## 5. 第三方服务

运营公司负责申请和持有：

- 短信服务商账号。
- 广告平台账号。
- 微信或支付宝商户号。
- 域名证书。
- 应用市场开发者账号。

开发者负责：

- 按服务商文档接入接口。
- 提供配置位置。
- 提供回调地址。
- 验证联调结果。
- 不长期保存运营方密钥。

## 6. APP 构建资料

应交付：

- Flutter SDK 版本要求。
- Android 构建命令。
- API 地址配置方式。
- 测试模式开关说明。
- APK 或 AAB 构建产物。

正式发包前必须确认：

- `API_BASE_URL` 指向正式 HTTPS API。
- `TEST_MODE=false`，除非明确是内测包。
- 应用名称、图标、包名、签名信息由运营方确认。
- 隐私政策和 SDK 清单已经准备。

## 7. 管理后台

应交付：

- 后台访问地址。
- 初始管理员账号创建方式。
- 用户账本说明。
- 广告收益说明。
- 提现审核流程说明。
- 系统配置说明。

不建议交付固定默认密码。管理员初始密码应由运营方在服务器上临时设置并立即保存到密码管理工具。

## 8. 验收项目

基础验收：

- 后端 `/health` 正常。
- 管理后台能登录。
- 手机号登录正常。
- 用户签到后后台账本同步。
- 转盘次数限制正常。
- 金币流水记录正常。
- 提现申请能进入后台审核。
- 广告收益页面能显示模拟或真实数据。

安全验收：

- Git 仓库无 `.env`。
- Git 仓库无私钥。
- Git 仓库无真实 token。
- Git 仓库无个人隐私信息。
- 生产后台使用 HTTPS。
- MySQL 不开放公网。
- 管理员密码不是默认弱密码。

运营验收：

- 提现规则已写清楚。
- 封号规则已写清楚。
- 隐私政策已覆盖手机号、设备信息、广告 SDK、提现信息。
- 用户协议已覆盖金币、提现、违规处理。
- 第三方 SDK 清单已准备。
