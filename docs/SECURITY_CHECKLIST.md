# 安全检查清单

本清单用于每次交付、部署、发包前检查。

## 1. 仓库安全

检查项：

- 不提交 `.env`。
- 不提交数据库备份。
- 不提交服务器 SSH 私钥。
- 不提交广告平台密钥。
- 不提交短信服务密钥。
- 不提交支付或提现商户密钥。
- 不提交真实用户手机号、身份证、银行卡、提现账号。

建议命令：

```bash
git status --short
git ls-files | grep -E '(^|/)\.env$|id_rsa|id_ed25519|\.pem$|\.key$'
grep -RInE 'BEGIN .*PRIVATE KEY|AKIA[0-9A-Z]{16}|sk-[A-Za-z0-9_-]{20,}|ghp_|github_pat_' . \
  --exclude-dir=node_modules --exclude-dir=dist --exclude-dir=build --exclude-dir=.git
```

## 2. 生产环境密钥

必须满足：

- `JWT_SECRET` 至少 32 字符。
- `ADMIN_JWT_SECRET` 至少 32 字符。
- 两个 JWT 密钥不能相同。
- 数据库密码不能使用弱密码。
- 管理员密码至少 12 位。
- 密钥只放在服务器 `.env` 或云厂商密钥管理服务中。

生成随机密钥：

```bash
openssl rand -base64 48
```

## 3. 服务器安全

必须满足：

- 服务器只开放 22、80、443。
- MySQL 不开放公网。
- SSH 禁止弱密码。
- 后台只使用 HTTPS。
- 腾讯云安全组限制来源 IP。
- 定期更新系统补丁。
- 定期备份数据库。

建议：

- 后期把 MySQL 迁移到腾讯云数据库。
- 给管理后台增加 IP 白名单或二次验证。
- 给 Nginx 和 PM2 配置日志轮转。

## 4. 接口安全

必须满足：

- 后台接口必须走管理员 JWT。
- 普通用户接口必须走用户 JWT。
- 广告奖励不能只相信客户端。
- 短信验证码必须限频。
- 提现必须人工审核。
- 金币变动必须写流水。
- 关键接口必须限流。

重点风险：

- 用户重复刷广告奖励。
- 同一设备注册多个账号刷金币。
- 同一 IP 批量登录或发短信。
- 广告回调被伪造。
- 提现接口被脚本批量提交。

## 5. 广告和金币安全

正式接广告 SDK 后必须做到：

- 广告平台回调必须校验签名。
- 回调 ID 必须去重。
- 同一用户每日广告奖励有上限。
- 同一设备每日广告奖励有上限。
- 金币奖励以服务端入账为准。
- 后台能看到广告收益和金币发放成本。

不要把“客户端播放完成”直接当作奖励依据。客户端可以作为展示反馈，但最终入账必须由后端确认。

## 6. 提现安全

正式运营前必须做到：

- 最低提现门槛。
- 每日提现次数限制。
- 每日提现金额限制。
- 新用户提现观察期。
- 异常用户冻结。
- 人工审核记录。
- 拒绝原因记录。

早期建议只做人工打款，不要一开始接自动打款。

## 7. 合规材料

上线前应准备：

- 用户协议。
- 隐私政策。
- 第三方 SDK 清单。
- 权限使用说明。
- 注销账号说明。
- 提现规则。
- 广告奖励规则。
- 反作弊和封号规则。

如果在中国大陆对外运营，还需要由运营公司办理网站备案、APP 备案以及应用市场所需材料。
