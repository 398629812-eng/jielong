# 第三方服务接入说明

本文档说明短信、广告 SDK、提现通道的接入边界。运营公司负责申请平台账号和资质，开发者负责按配置接入系统。

## 1. 接入原则

- 平台账号归运营公司所有。
- 合同、结算、发票、税务由运营公司负责。
- 密钥只配置在服务器环境变量或云厂商密钥服务中。
- Git 仓库不保存任何真实密钥。
- 所有第三方回调都要校验签名和去重。

## 2. 短信服务

可选服务商：

- 腾讯云短信
- 阿里云短信
- 华为云短信
- 其他合规短信服务商

运营公司需要提供：

- 短信服务商名称。
- AccessKey ID。
- AccessKey Secret。
- 短信签名。
- 验证码模板 ID。
- 国内短信发送区域和价格。

后端需要实现：

- 发送验证码接口。
- 验证码保存和过期。
- 手机号发送限频。
- IP 发送限频。
- 验证错误次数限制。
- 短信发送日志。

建议规则：

- 同一手机号 60 秒内只能发送一次。
- 同一手机号每天最多 5 到 10 次。
- 同一 IP 每天最多 30 到 50 次。
- 验证码 5 分钟过期。

## 3. 广告 SDK

可选广告平台：

- 穿山甲
- 优量汇
- 快手联盟
- 百度联盟
- 聚合广告平台，如 TopOn、TradPlus、GroMore

运营公司需要提供：

- 广告平台账号。
- App ID。
- 广告位 ID。
- 激励视频广告位 ID。
- 插屏广告位 ID。
- 信息流广告位 ID。
- 服务端回调地址配置权限。
- 回调签名密钥。
- 结算规则和报表口径。

推荐奖励链路：

```text
用户点击看广告
APP 调用 SDK 播放广告
广告平台确认有效播放
广告平台回调后端
后端校验签名和回调 ID
后端写入 ad_records
后端写入 gold_records
用户刷新金币余额
```

正式环境不要只靠客户端上报奖励。客户端可能被篡改，必须以广告平台服务端回调为最终入账依据。

后台建议保留以下字段：

- 用户 ID
- 手机号或用户名
- 广告平台
- 广告类型
- 广告位 ID
- 回调 ID
- 收益金额
- 校验状态
- 创建时间

当前系统已经预留广告收益记录结构，后续接真实 SDK 时可继续扩展签名校验和平台适配层。

## 4. 提现通道

可选方式：

- 人工审核后手动转账。
- 微信商户转账。
- 支付宝转账。
- 合规第三方代付。

早期建议使用人工审核，不建议直接自动打款。

运营公司需要提供：

- 商户主体。
- 商户号。
- 打款方式。
- 结算账户。
- 风控要求。
- 提现规则。
- 客服联系方式。

系统需要保证：

- 提现申请写入数据库。
- 管理后台人工审核。
- 审核通过后再打款。
- 审核拒绝有原因。
- 用户余额和提现记录可追溯。
- 异常用户可冻结。

## 5. 配置建议

后续可在 `.env` 中增加类似配置：

```env
SMS_PROVIDER=tencent
SMS_ACCESS_KEY_ID=
SMS_ACCESS_KEY_SECRET=
SMS_SIGN_NAME=
SMS_TEMPLATE_CODE=

AD_PROVIDER=pangle
AD_CALLBACK_SECRET=
AD_APP_ID=

WITHDRAW_PROVIDER=manual
```

真实密钥只填写在服务器 `.env`，不提交到 Git。

## 6. 联调验收

短信验收：

- 真实手机号能收到验证码。
- 错误验证码不能登录。
- 验证码过期后不能登录。
- 高频发送会被限制。

广告验收：

- APP 能加载激励视频。
- 播放完成后平台能回调后端。
- 同一回调不会重复发金币。
- 后台广告收益页面能看到记录。
- 用户账本能看到金币变动。

提现验收：

- 用户可提交提现申请。
- 后台能审核通过或拒绝。
- 金币扣减和提现记录一致。
- 风险用户不能提现。
